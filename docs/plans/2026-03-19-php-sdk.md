# PHP SDK implementation plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to
> implement this plan task-by-task.

**Goal:** Build a standalone PHP SDK in `/Users/yanming/sol/sdk-php` that
matches the Go SDK's behavior for tracking, signing, A/B evaluation, sticky
results, metadata loading, and exposure logging, while exposing a PHP-native
API.

**Architecture:** The PHP repository mirrors the Go SDK's behavioral layers.
Tracking, signing, and A/B evaluation stay close to the Go implementation so
that test fixtures and edge cases can be ported directly. The public API remains
PHP-friendly, but the underlying domain objects and algorithms preserve the Go
SDK's semantics.

**Tech Stack:** PHP 8.2+, Composer, PHPUnit, PHPStan, PSR-4 autoloading,
ext-json, ext-hash, ext-openssl, ext-mbstring, and a lightweight HTTP client
based on cURL.
